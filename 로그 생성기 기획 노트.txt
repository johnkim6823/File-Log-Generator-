가설
	인원의 삭제, 변동에 따른 변경사항은 고려하지 않는다. 
	- 퇴직자나 보직이 완전히 바뀐 직원이 이전 파일을 보는 것은 간단한 규칙으로 잡아낼 수 있음으로 고려하지 않는다.
	(현재 직원들의 로그를 기록하는 것에 집중)

	중앙 파일 서버 존재 + endpont monitoring이 이루어지는 상황을 가정한다.
	- 중앙 파일 서버가 존재하여 파일의 소유자 추적과 공유 요청이 이루어질 수 있는 상황 가정 
	- endpont monitoring이 존재하여 작업자의 로그가 정상적으로 수집되는 상황 가정

	기본적으로는 타 부서 간 파일 공유, 전송은 잘 일어나지 않지만, 
	부서간 협업 시, 또는 (전직원 또는 일정 부서) 공유용으로 생성된 파일의 경우 타 부서에게도 파일이 공유 될 수 있다.
	- 공유용으로 생성된 파일 예제: 간단한 회사 생활 가이드, 특정 부서들에서 쓰이는 작업 도구 파일들 등


config들은 json 형태로 수정 + 변경 가능해야 함



전역 설정
	사이클당 시간, 

파일
	DB에 저장됨(No-sql?)
	파일 이름, 등급, 소유자, 읽기 권한 소유자, 쓰기 권한 소유자, 실행 권한 소유자, 등록 시간, 최종 갱신 시간 등이 저장되어야 함
	class 형태로도 정의가 필요(각 직원 pc에 로컬로 저장된 파일을 구현하기 위해)

부서
	직원들의 집합
	각 부서별 부서원의 작업 패턴도 포함하여 선언됨
	- ex: 2등급 직원은 X% 확률로 X 홛동을 하고 ... (json 형태로는 'action' : [['write', 0.25], ...]  같은 느낌)
	확률적으로 부서간 협업 또는 부서내 협업 이벤트 발생 가능

직원
	일반적으로 부서에서 정의된 행동 패턴을 가져가나, 세부 지정도 가능


	기본 행동:
		파일 생성
		파일 읽기
		파일 쓰기 : 기존 가지고 있는 파일 중 하나 업데이트(최근 것만?)
		파일 실행(실행파일이라면)

		파일 전송
		파일 공유 요청
		파일 공유 요청 수락
		파일 공유 요청 거부

	복합행동(내부적으로만 정의됨, 부서 이벤트 관련)
		파일 취합 및 정리
		- 상위 직원이 하위 부서원이 생성한 파일을 일정량 수집하여 정리 파일을 생성함
		- 충분한 파일이 모일떄까지 대기를 하다가, 파일이 모인 후 상위 단계 파일(정리 파일)을 생성

		파일 상신
		- 파일을 생성한 후 최종 작업물을 상신 

		하위 직원은 이전 자료나, 기타 자료를 수집한 후 정리해서 파일을 만들고 상신, 
		중간 직원은 상신된 파일을 수집하여 정리한 후 상신
		최상위 직원은 단계를 거쳐 올라온 파일을 수집
		(여러 단계로 상신과 정리가 이루어질 수 있음)

(잠재적으로)악의적인 직원
	- 직원 class를 상속한 클래스
	- 일반적인 상태에서는 정상 직원과 동일
	- 특정 조건이 활성화 되면 정해진 시나리오의 악의적 행동 실행
	- 한명이 악의적으로 변할수도 있고 여려명이 같이 변할 수 있다.
	(모든 직원이 악의적인 직원이 될 가능성이 있어야 함으로 모든 직원은 해당 클래스로 생성될 것임)



파일의 소유자:
	(문서 파일의 경우)파일의 읽기와 쓰기 권한을 가진 모든 직원

	(실행 파일의 경우)파일의 (읽기, 쓰기 그리고) 실행 권한을 가진 모든 직원(고려중)


파일 공유:
	요청 -
	파일을 db에 검색했을 때 찾은 파일의 소유자에게 요청
	요청은 요청하는 직원의 요청 대기 큐에 들어감

	수락 및 거부 -
	요청 대기 큐에 요청이 있다면 수락 또는 거부 수행
	확률적으로 요청을 수락하거나 거부
	- 경우에 따라 확률이 변화해야 할텐데 구현 방식은?
	- 요청자의 부서, 협업 여부, 요청 파일 등급 등에 따라서 확률이 달라져야 함



일반적인 흐름:
	1. 부서 순회: 부서 내 이벤트 발생 여부 결정
	- 이미 이벤트 중이라면 스킵
	- 특정 확률에 따라 이벤트 발생
	- 발생 시 해당 부서원에게 작업 부여

	2. 작업 수행자 선택: 해당 사이클에 작업할 직원 선택
	- 직원과 횟수를 선택한 후 랜덤으로 순서 설정(작업 큐 생성)
	- 직원 전체를 한번 순회하여 작업 수행자를 선택

	3. 작업 수행
	- 생성한 작업 큐 순서대로 작업 수행
	- 각 작업자가 랜덤하게 수행해야할 작업을 수행
	- 작업 별 시간 간격은 어떻게?

악의적인 행동을 위한 추가 흐름:
	0. 악의적인 사용자 활성화
	- 랜덤하게 악의적인 사용자 활성화



기타 노트:
	해커가 취약점을 이용해 파일을 수집하는 경우는 어떻게 나타날 수 있을까요?
	- 어러 컴퓨터/계정에서 사칭한 계정으로 파일을 전송하는 방식이 될 겁니다.



	해당 프로젝트를 기획하면서 느끼는 것이지만, 마치 마피아 게임을 기획하는 느낌이 든다.
	각 직원들은 npc이고 각 npc의 기본 ai를 작업하는 느낌이 든다.
